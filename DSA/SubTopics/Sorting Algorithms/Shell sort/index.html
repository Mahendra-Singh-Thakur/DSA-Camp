Shell Sort

Shell Sort is a generalization of Insertion Sort that allows the exchange of items that are far apart. The idea is to arrange the list of elements so that, starting anywhere, taking every \(h\)-th element produces a sorted list. By using an increment sequence \(h_1, h_2, ..., h_k\), we can ensure that the elements are eventually h-sorted for some value of \(h\).

Time Complexity:

The time complexity of Shell Sort heavily depends on the gap sequence used. In this implementation, we are using a gap sequence of \(n/2\), \(n/4\), \(n/8\), and so on until the gap becomes 1. The average time complexity of Shell Sort with this gap sequence is \(O(n \log n)\). However, it can perform much better than this in practice. The worst-case time complexity is still \(O(n^2)\), but because the gaps help to partially sort the array, the average time is significantly better than traditional insertion sort for larger datasets.

Space Complexity:

The space complexity of Shell Sort is \(O(1)\) because it only uses a constant amount of extra space for variables like loop counters and temporary variables.
