Selection Sort

Selection sort is a simple sorting algorithm that divides the input list into two parts: a sorted sublist and an unsorted sublist. Initially, the sorted sublist is empty, and the unsorted sublist contains all elements. The algorithm selects the smallest element from the unsorted sublist and swaps it with the leftmost unsorted element, moving the sublist boundaries one element to the right.

Steps to Perform Selection Sort:

  1. Find the smallest element in the unsorted sublist.
  2. Swap it with the leftmost unsorted element.
  3. Move the sublist boundaries one element to the right.
  4. Repeat steps 1-3 until the entire array is sorted.

Time Complexity:

Selection sort is a simple comparison-based sorting algorithm that divides the input array into two parts: a sorted subarray and an unsorted subarray. It repeatedly selects the smallest (or largest, depending on the sorting order) element from the unsorted subarray and swaps it with the first element of the unsorted subarray.

The time complexity of selection sort is O(n^2), where n is the number of elements in the input array. This time complexity remains the same regardless of the distribution of elements in the input array. This is because, in each iteration of the algorithm, it needs to find the minimum (or maximum) element from the unsorted subarray, which requires scanning through the unsorted portion of the array.

Selection sort performs better than bubble sort but still has poor performance on large datasets or datasets with a large number of inversions.

Space Complexity:

Selection sort is an in-place sorting algorithm, meaning it doesn't require additional space beyond the input array itself. Therefore, the space complexity of selection sort is O(1), indicating constant space usage, regardless of the size of the input array.

Selection Sort Algorithm

                        // Input: array - the array to be sorted
                        Procedure selectionSort(array)

                            DECLARE n = LENGTH(array) // Length of the array
                            DECLARE minIndex // Index of the minimum element

                            // Traverse the array
                            FOR (i = 0) TO (n - 1) DO
                                minIndex = i // Assume the current element is the minimum

                                // Find the index of the minimum element in the unsorted sublist
                                FOR (j = i + 1) TO (n - 1) DO
                                    IF array[j] < array[minIndex] THEN
                                        minIndex = j
                                    END IF
                                END FOR

                                // Swap the minimum element with the current element
                                IF i != minIndex THEN
                                    SWAP(array[i], array[minIndex])
                                END IF
                            END FOR
                        End Procedure
